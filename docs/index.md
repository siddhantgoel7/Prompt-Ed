# Project Requirements

## Executive Summary
This project aims to develop a real-time, AI-assisted classroom discussion tool designed for discussion-based university lectures in the Department of Pharmacology. The system combines live speech-to-text transcription ("hearing"), presentation content analysis ("seeing"), and AI-driven prompt generation to support interactive, anonymous student participation during lectures.

Instructors use the platform to create and manage lessons, generate discussion prompts from live lecture content, and collect anonymous student responses via QR code or PIN-based access. Students participate through a lightweight web interface without requiring authentication, ensuring privacy and ease of access. After each session, the system automatically aggregates prompts, responses, transcripts, and analytics for instructor review and export.

The primary quality goals of the system are convenience, accuracy, and security, ensuring minimal disruption during live lectures while maintaining reliable AI output and protecting user data.

## Project Glossary
- **Lesson** – A single lecture session created and managed by an instructor.
- **Discussion Prompt** – A question generated by AI or manually created by an instructor to initiate student discussion.
- **Session** – A live instance of a lesson during which students can join and participate.
- **STT (Speech-to-Text)** – Real-time transcription of instructor audio during a lecture.
- **QR Code / PIN** – Mechanisms used by students to join a live lesson.
- **Anonymous Response** – A student submission that is not linked to any identifiable user data.
- **Prompt State** – The current lifecycle state of a discussion (e.g., open or closed).
- **Context Builder** – A system layer that controls what lecture content is sent to the AI to reduce hallucinations and cost.
