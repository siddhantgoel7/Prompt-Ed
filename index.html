    Project Requirements

    Executive Summary

    This project aims to develop a real-time, AI-assisted classroom discussion tool designed for discussion-based university lectures in the Department of Pharmacology. The system combines live speech-to-text transcription ("hearing"), presentation content analysis ("seeing"), and AI-driven prompt generation to support interactive, anonymous student participation during lectures.

    Instructors use the platform to create and manage lessons, generate discussion prompts from live lecture content, and collect anonymous student responses via QR code or PIN-based access. Students participate through a lightweight web interface without requiring authentication, ensuring privacy and ease of access. After each session, the system automatically aggregates prompts, responses, transcripts, and analytics for instructor review and export.

    The primary quality goals of the system are convenience, accuracy, and security, ensuring minimal disruption during live lectures while maintaining reliable AI output and protecting user data.

    Project Glossary

    • Lesson – A single lecture session created and managed by an instructor.
    • Discussion Prompt – A question generated by AI or manually created by an instructor to initiate student discussion.
    • Session – A live instance of a lesson during which students can join and participate.
    • STT (Speech-to-Text) – Real-time transcription of instructor audio during a lecture.
    • QR Code / PIN – Mechanisms used by students to join a live lesson.
    • Anonymous Response – A student submission that is not linked to any identifiable user data.
    • Prompt State – The current lifecycle state of a discussion (e.g., open or closed).
    • Context Builder – A system layer that controls what lecture content is sent to the AI to reduce hallucinations and cost.

    User Stories

    Account Management

    US 1.01 – Create Account (Must Have)

    As an instructor, I want to be able to create an account on the platform, so that I can access the system and manage my teaching materials.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor without an account WHEN they click "Sign in with Google" and select their UAlberta email THEN they are redirected to UAlberta SSO login

    2. GIVEN an instructor WHEN they complete UAlberta SSO authentication successfully THEN their account is automatically created and they are logged into the system

    3. GIVEN an instructor WHEN they attempt to sign in with a non-UAlberta Google account THEN authentication is blocked with error message "Please use your UAlberta email"

    4. GIVEN an instructor WHEN they sign in with UAlberta email for the first time THEN they are directed to the instructor dashboard

    US 1.02 – Login via UAlberta Authentication (Must Have)

    As a instructor, I want to be able to login via UAlberta authentication, so that I can access my materials and have instructor privileges.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor with existing account WHEN they click "Sign in with Google" and authenticate via UAlberta SSO THEN they successfully login and access the instructor dashboard

    2. GIVEN an instructor WHEN they fail UAlberta SSO authentication THEN login fails and they see error message "Authentication failed"

    3. GIVEN an instructor WHEN they close and reopen the application within session timeout THEN they remain logged in without re-authenticating

    US 1.03 – Logout Securely (Must Have)

    As a instructor, I want to be able to logout securely, so that I can protect my account when finished.

    Points assigned: 2

    Acceptance Tests:

    1. GIVEN a logged-in instructor WHEN they click "Logout" from the profile page THEN they are logged out and session is terminated

    2. GIVEN an instructor WHEN they logout THEN they are redirected to the login page

    3. GIVEN a logged-out instructor WHEN they attempt to access protected pages via URL THEN they are redirected to login page

    4. GIVEN an instructor WHEN they logout THEN all session data is cleared from the browser

    US 1.04 – View Only Own Past Lessons (Could Have)

    As a instructor, I want only I to be able to view my past lessons and details, so that my lessons remain private.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor WHEN they access their dashboard THEN they see only their own past lessons, not other instructors' lessons

    2. GIVEN an instructor WHEN they attempt to access another instructor's lesson via direct URL THEN access is denied with 403 Forbidden error

    3. GIVEN an instructor WHEN they view lesson details THEN only data from their own lessons is displayed

    Lesson Management

    US 1.05 – Create New Lesson (Must Have)

    As a instructor, I want to be able to create a new lesson, so that I can prepare content for a lecture.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor WHEN they click "Create Lesson" and provide a title THEN a new lesson is created with inactive status

    2. GIVEN an instructor WHEN they create a lesson without providing a title THEN the system generates a default title with current date/time

    3. GIVEN an instructor WHEN they create a new lesson THEN it appears in their lesson list

    4. GIVEN an instructor WHEN they create a lesson without uploading files THEN the lesson is still created successfully

    US 1.06 – Start a Lesson (Must Have)

    As a instructor, I want to be able to start a lesson, so that students can join and I can begin teaching.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor with a created lesson WHEN they click "Start Lesson" THEN the lesson status changes to active

    2. GIVEN an instructor WHEN they start a lesson THEN a unique PIN code is generated and displayed

    3. GIVEN an instructor WHEN they start a lesson THEN a unique QR code is generated and displayed

    4. GIVEN an instructor WHEN the lesson is active THEN students can join using the PIN or QR code

    US 1.07 – Cancel a Lesson (Won't Have)

    As a instructor, I want to be able to cancel a lesson, so that I can abort lesson creation if needed.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor creating a lesson WHEN they click "Cancel" THEN the lesson is not saved to their lesson list

    2. GIVEN an instructor WHEN they cancel lesson creation THEN they are returned to the appropriate page

    3. GIVEN an instructor WHEN they cancel a lesson THEN any uploaded files for that lesson are discarded Note: We don't need cancel creation. Just delete the lesson or something.

    US 1.08 – Delete Past Lesson (Should Have)

    As a instructor, I want to be able to delete a past lesson, so that I can remove outdated or incorrect lessons.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor WHEN they select a past lesson and click "Delete" THEN a confirmation dialog appears

    2. GIVEN an instructor WHEN they confirm deletion THEN the lesson is permanently removed from the system

    3. GIVEN an instructor WHEN they delete a lesson THEN it no longer appears in their lesson list

    4. GIVEN an instructor WHEN they delete a lesson THEN all associated data (prompts, responses, transcripts, files) is permanently deleted

    US 1.09 – End/Leave a Lesson (Must Have)

    As a instructor, I want to be able to end/leave a lesson, so that the lesson is closed.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor with active lesson WHEN they click "end/leave Lesson" THEN the lesson status changes to inactive

    2. GIVEN an instructor WHEN they end/leave a lesson THEN students cannot submit new responses

    3. GIVEN an instructor WHEN they end/leave a lesson THEN students are notified that the lesson has ended

    4. GIVEN an instructor WHEN they end/leave a lesson THEN all lesson data (prompts, responses, transcripts) is saved automatically

    5. GIVEN an instructor WHEN they end/leave a lesson THEN files, prompts, questions, and analytics are ready for export

    US 1.10 – Automatic Lesson Save on End (Must Have)

    As a instructor, I want the lesson to be saved automatically when I end it, so that transcripts, prompts, and responses are preserved.

    Points assigned: 5

    Acceptance Tests:

    1. GIVEN an instructor WHEN they end a lesson THEN all lesson data is automatically saved without manual action required

    2. GIVEN an instructor WHEN they end a lesson THEN all discussion prompts with timestamps are preserved

    3. GIVEN an instructor WHEN they end a lesson THEN all student responses with submission times are preserved

    4. GIVEN an instructor WHEN they end a lesson and STT was used THEN all transcripts are preserved

    5. GIVEN an instructor WHEN they access a saved lesson THEN they can view all preserved data

    US 1.11 – Resume Unfinished Lesson (Could Have)

    As a instructor, I want to be able to resume a previously unfinished lesson, so that I can continue where I left off.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor WHEN they view their dashboard THEN they can see a list of unfinished lessons separate from finished lessons

    2. GIVEN an instructor WHEN they select an unfinished lesson and click "Resume" THEN the lesson opens with all previous data intact

    3. GIVEN an instructor WHEN they resume a lesson THEN previously uploaded files are available

    4. GIVEN an instructor WHEN they resume a lesson THEN previously created discussions are available

    US 1.12 – Reconnect After Connection Failure (Must Have)

    As a instructor, I want to be able to reconnect if my connection fails, so that I can continue my lesson where I left off.

    Points assigned: 5

    Acceptance Tests:

    1. GIVEN an instructor with active lesson WHEN connection is lost THEN they can click "Reconnect" to rejoin

    2. GIVEN an instructor WHEN they reconnect to active lesson THEN the lesson state (current discussion, student count) is preserved

    3. GIVEN an instructor WHEN they reconnect THEN student responses submitted during disconnection are visible

    4. GIVEN an instructor WHEN they reconnect THEN they see a message confirming successful reconnection

    US 1.13 – Automatic Data Saving at Regular Intervals (Must Have)

    As a instructor, I want to have my lesson data automatically saved at regular intervals, so that progress is not lost if something goes wrong.

    Points assigned: 5

    Acceptance Tests:

    1. GIVEN an instructor with active lesson WHEN they are teaching THEN lesson data is automatically saved at regular intervals

    2. GIVEN an instructor WHEN connection is lost THEN discussion prompts created before disconnection are preserved

    3. GIVEN an instructor WHEN connection is lost THEN student responses submitted before disconnection are preserved

    4. GIVEN an instructor WHEN they reconnect after unexpected disconnection THEN auto-saved data is recoverable

    US 1.14 – View Past Lessons and Details (Must Have)

    As a instructor, I want to be able to view past lessons and its details, so that I can review transcripts, prompts, and responses.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor WHEN they access their dashboard THEN they see a list of all past lessons

    2. GIVEN an instructor WHEN they select a past lesson THEN they can view the lesson details page

    3. GIVEN an instructor WHEN viewing past lesson details THEN they see all discussion prompts that were published

    4. GIVEN an instructor WHEN viewing past lesson details THEN they see all student responses organized by prompt

    5. GIVEN an instructor WHEN viewing past lesson details and STT was used THEN they see all transcripts

    US 1.15 – Helpful Error Messages (Should Have)

    As a instructor, I want to see helpful error messages when something goes wrong, so that I can troubleshoot issues.

    Points assigned: 2

    Acceptance Tests:

    1. GIVEN an instructor WHEN file upload fails THEN they see a clear error message indicating the failure reason

    2. GIVEN an instructor WHEN AI prompt generation fails THEN they see error message "Prompt generation failed. Please try again or enter manually"

    3. GIVEN an instructor WHEN connection is lost THEN they see error message "Connection lost. Attempting to reconnect..."

    4. GIVEN an instructor WHEN an error occurs THEN the error message is displayed prominently and clearly

    US 1.49 – Add Course to Courselist (Must Have)

    As a instructor, I want to be able to add a course which contains all lessons specific to that course to the courselist, so that I can organize my lessons by course.

    Points assigned: 2

    Acceptance Tests:

    1. GIVEN an instructor WHEN they click "Create Course" THEN they can enter a course name and optional description

    2. GIVEN an instructor WHEN they create a course THEN it appears in their courses list

    3. GIVEN an instructor WHEN they create or edit a lesson THEN they can assign it to an existing course

    4. GIVEN an instructor WHEN they view a course THEN they see all lessons assigned to that course

    US 1.50 – Delete Course from Courselist (Must Have)

    As a instructor, I want to be able to delete a course which contains all the lessons specific to that course to the courselist, so that I can remove courses I no longer need.

    Points assigned: 2

    Acceptance Tests:

    1. GIVEN an instructor WHEN they select a course and click "Delete" THEN a confirmation dialog appears warning about lesson deletion

    2. GIVEN an instructor WHEN they confirm course deletion THEN the course and all its lessons are permanently removed

    3. GIVEN an instructor WHEN they confirm course deletion THEN all files and metadata associated with lessons in that course are permanently deleted

    Content Upload & Audio Capture

    US 1.16 – Upload Files (PDF, PPTX) (Must Have) [5 pts]

    As an instructor
    I want to be able to upload files (PDF, PPTX)
    So that I can provide sources for discussion prompts

    Acceptance Tests: - GIVEN an instructor WHEN they upload a PDF file within size limit THEN the file is accepted and appears in the uploaded files list - GIVEN an instructor WHEN they upload a PPTX file within size limit THEN the file is accepted and appears in the uploaded files list - GIVEN an instructor WHEN they attempt to upload an unsupported file type (e.g., .docx, .txt) THEN upload fails with error message "Unsupported file type. Please upload PDF or PPTX" - GIVEN an instructor WHEN they upload files THEN they can see a list of all uploaded files with names and types - GIVEN an instructor WHEN they upload multiple files THEN the system accepts up to 5 files per lesson

    US 1.17 – Activate/Deactivate STT Transcript Capture (Must Have)

    As a instructor, I want to be able to activate and deactivate STT transcript capture, so that I can control when my spoken content is recorded.

    Points assigned: 5

    Acceptance Tests:

    1. GIVEN an instructor with active lesson WHEN they click "Start Recording" THEN the recording period begins and STT transcript capture activates

    2. GIVEN an instructor WHEN recording period is active THEN their speech is transcribed in real-time

    3. GIVEN an instructor WHEN they click "Stop Recording" THEN the recording period ends and STT transcript capture deactivates

    4. GIVEN an instructor WHEN they stop recording THEN the transcript is sent to AI for prompt generation

    5. GIVEN an instructor WHEN recording period is active THEN they can toggle STT on and off multiple times during the lesson

    AI Prompt Generation

    US 1.18 – Trigger AI Prompt Generation (Must Have)

    As a instructor, I want to be able to trigger AI prompt generation, so that the system creates discussion questions based on my content.

    Points assigned: 5

    Acceptance Tests:

    1. GIVEN an instructor with uploaded files WHEN they click "Generate Prompts" THEN AI processes the files and generates discussion prompts

    2. GIVEN an instructor with completed transcript WHEN recording period ends THEN AI automatically generates prompts from the transcript and files

    3. GIVEN an instructor WHEN AI is generating prompts THEN they see a loading indicator

    4. GIVEN an instructor WHEN AI completes generation THEN at least 3 discussion prompts are displayed

    US 1.19 – Review and Select AI Generated Prompts (Must Have)

    As a instructor, I want to be able to review and select AI generated prompts, so that I can choose the most appropriate discussion question.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor WHEN AI generates prompts THEN all generated prompts are displayed for review

    2. GIVEN an instructor WHEN they view generated prompts THEN they can select one prompt from the options

    3. GIVEN an instructor WHEN they select a prompt THEN it is highlighted or visually marked

    4. GIVEN an instructor WHEN they select a prompt THEN they can change their selection before publishing

    US 1.20 – Edit AI-Generated Prompts (Could Have)

    As a instructor, I want to be able to edit AI-generated prompts before publishing, so that I can refine the wording.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor with selected AI prompt WHEN they click "Edit" THEN the prompt text becomes editable

    2. GIVEN an instructor WHEN they modify the prompt text THEN the changes are saved

    3. GIVEN an instructor WHEN they edit a prompt THEN they can see both original and edited versions

    4. GIVEN an instructor WHEN they edit a prompt THEN they can revert to the original AI-generated text

    US 1.21 – Manually Enter Prompt (Should Have)

    As a instructor, I want the option to manually enter a prompt, so that I can continue the discussion even if AI fails.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor WHEN they click "Create Custom Prompt" THEN a text input field appears for manual entry

    2. GIVEN an instructor WHEN they manually enter a prompt THEN it functions the same as AI-generated prompts

    3. GIVEN an instructor WHEN they create a manual prompt THEN they can publish it without triggering AI generation

    4. GIVEN an instructor WHEN AI generation fails THEN they see option to manually enter a prompt

    US 1.22 – Set AI Prompt Generation Preferences (Should Have)

    As a instructor, I want to be able to set preferences for how AI generates prompts, so that prompts align with my teaching style.

    Points assigned: 5

    Acceptance Tests:

    1. GIVEN an instructor WHEN they access AI settings THEN they can configure prompt generation preferences

    2. GIVEN an instructor WHEN they set preferences THEN AI uses these preferences in future prompt generations

    3. GIVEN an instructor WHEN they change preferences THEN the new settings are saved and applied to subsequent generations

    US 1.23 – Generate Multiple Choice, Short Answer, and Long Answer Prompts (Must Have)

    As a instructor, I want to be able to generate multiple choice, short answer, and long answer type discussion prompts, so that I can vary discussion formats.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor WHEN they trigger AI generation THEN they can specify prompt type (multiple choice, short answer, or long answer)

    2. GIVEN an instructor WHEN they select multiple choice format THEN AI generates a question with answer options

    3. GIVEN an instructor WHEN they select short answer or long answer format THEN AI generates an open-ended question

    4. GIVEN an instructor WHEN they manually create a prompt THEN they can choose between multiple choice, short answer, or long answer formats

    US 1.24 – Regenerate AI Prompts (Must Have)

    As a instructor, I want to be able to regenerate AI prompts if I'm not satisfied, so that I can get alternative suggestions.

    Points assigned: 2

    Acceptance Tests:

    1. GIVEN an instructor with AI-generated prompts WHEN they click "Regenerate" THEN AI generates a new set of prompts

    2. GIVEN an instructor WHEN they regenerate prompts THEN the previous prompts are replaced with new ones

    3. GIVEN an instructor WHEN they regenerate THEN they see a loading indicator while AI generates new prompts

    Discussion Management

    US 1.25 – Create Multiple Discussions Within One Lesson (Must Have)

    As a instructor, I want to be able to create multiple discussions within one lesson, so that I can facilitate ongoing engagement throughout my lecture.

    Points assigned: 3

    Acceptance Tests:

    1. GIVEN an instructor with active lesson WHEN they publish a discussion prompt THEN they can create additional discussions in the same lesson

    2. GIVEN an instructor WHEN they create multiple discussions THEN each discussion is tracked separately
